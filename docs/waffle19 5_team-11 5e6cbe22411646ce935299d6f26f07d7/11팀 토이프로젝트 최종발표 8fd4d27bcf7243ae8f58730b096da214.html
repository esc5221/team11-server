Backend

  • auto_lecture_adder (자랑하고 싶지는 않음)
    • 수강신청 사이트에서 강의들 정보를 csv 파일로 긁어온다
    • 모든 강의에 대해 각각 PUT, POST 명령들을 보내 강의 정보를 추가, 갱신하는 프로그램 😅
  • 시간표는 자랑할만함
    • Lecture와 SubjectProfessor 두 개의 모델을 통해 강의 데이터를 표현함.
      • “과목명-교수" 정보를 담는 SubjectProfessor 모델을 만들고
      • 학기 별로 SubjectProfessor에 매칭되는 Lecture 모델을 생성
      • Review(강의평), Information(시험정보)을 SubjectProfessor에 연결하여 해당 강좌에 대한 전체 학기의 강의평과 시험 정보를 담을 수 있도록 했음.
    • Lecture에 quota(정원)과 people(담은 인원)을 보여줄 수 있도록 함.
      • 특정 lecture를 자신의 시간표에 담은 사용자의 수를 담은 인원에 표시함
    • CustomLecture을 통해 시간표에 강의를 넣음.
      • Schedule은 자신이 만든 시간표들을 나타냄
      • Lecture는 학기 별로 존재하는 모든 강의들을 가리킴
      • CustomLecture는 ‘특정 시간표와 Lecture의 매칭’을 가리킴
      • CustomLecture는 실제 존재하는 Lecture를 기반으로(정규 학기) 할 수도 있고, Lecture 데이터에 존재하지 않는 새로운 강의를 정의해서(계절 학기) 사용할 수도 있음.
  • lambda expression과 Q object를 활용해 코드 재사용성 증가

    → has_subcomments 여부가 각각의 객체에 반복적으로 사용되어 함수화

Frontend

  • 푸시 알림 / 알림함
    • Google Firebase Cloud Messaging 서비스 사용
      • 각 앱마다 FCM 토큰이라는 것이 존재
    • 푸시 알림을 받는 과정
      1. 앱을 설치할 때 FCM Token 생성
      1. 로그인을 성공하면 토큰을 서버로 보냄
      1. 내 게시글에 (대)댓글이 달리거나 쪽지를 받을 때 서버에서 토큰을 이용해 기기에 푸시 알림을 보냄
      1. 클라이언트에서 푸시 알림을 띄움
    • 알림함
      • (대)댓글이 달릴 때 날아오는 알림들은 메인 화면 4번 째 탭에 알림함에 모아둠
      • 알림 클릭하면 해당 게시물로 들어가짐
      • 서버로부터 내가 읽지 않은 알림의 개수(unread_count)를 badge로 띄움

  • 쪽지
    • 게시판에 글/댓글 쓴 사람한테 쪽지를 시작할 수 있음
    • 게시글과 비슷한 방식으로 구현 (RecyclerView)
    • 5번째 탭에 들어가면 쪽지방(message_room) 리스트를 볼 수 있음 → 클릭하면 각 쪽지방에 해당하는 쪽지를 볼 수 있음
    • 해당 activity와 fragment가 화면에 보여질 때마다 새로고침을 위해 onStart와 onResume을 override 함.
    • 안 읽은 쪽지가 있다면 하단에 badge로 표시함

  • 시간표
    • 각 학기의 시간표 생성, 삭제 가능
    • 서버에서 받아오는 강의를 추가할 수 있음
      • 다양한 검색 필터(학과, 검색어, 정렬, 학년, 구분, 학점) 적용 가능
    • 직접 시간대를 정해서 강의를 추가할 수 있음
      • 월~금 24시간 모두 시간 지정 가능
    • 기본 표시되는 시간대는 09~16시이지만, 강의의 시간대에 따라 동적으로 높이가 변하는 시간표를 구현 (에타 사양 반영)
    • 시간표의 셀을 눌러서 강의평, 강의계획서를 볼 수 있음
    • 강의 삭제, 수정, 약칭 지정, 메모 등의 기능이 구현됨

  • 강의평
    • ReviewActivity
      • 내 강의평: 내 default 시간표에 있는 과목들을 recyclerview에 불러옴. ‘평가하기’ 버튼을 통해 해당 과목의 강의정보화면으로 넘어갈 수 있음(LectureInfoActivity)
      • 최근 강의평: 전체 강의평목록을 최신순으로 가져와 보여줌. pagination 적용. 아이템을 눌러 해당 과목의 강의정보화면으로 넘어갈 수 있음.
      • 강의 검색: 키워드를 입력하고 엔터를 누르거나 검색 버튼을 누르면 해당하는 강의목록을 보여줌. pagination 적용됨. 강의 목록을 불러올 때는 서버에 subject_name=keyword query로 한번 요청하고, professor=keyword query로 다시 요청하여 두 경우의 response를 병합함. 아이템을 눌러 해당 과목의 강의정보화면으로 넘어갈 수 있음.
    • LectureInfoActivity
      • tableLayout을 이용해 강의 정보 표시
      • 강의평: 강의평이 없을 경우 없다고 표시. 강의평이 있으면 recylerview에 강의평 목록을 불러옴. 평균 별점은 0.01 단위로 별이 채워지도록 함.
      • 강의평 작성: 서버의 강의정보에서 개설학기 목록을 spinner에 넣어 보여줌. chipgroup을 통해 세부적인 평가를 선택할 수 있도록 하였고 선택된 칩의 id를 추출하여 서버에 전달함.
  • 시험정보
    • tableLayout을 이용해 시험 정보 표시. 강의평과 유사
    • 시험정보 작성: 다중선택이 가능한 chipgroup을 이용해 문제유형을 선택할 수 있도록 하였고 선택된 칩의 text를 추출하여 서버에 전달함. 문제예시는 개행을 기준으로 list화하여 서버에 전달함.

  • Django Model에 배열을 이용해 값을 저장할 수 없어 불편함이 있었음
    • String을 이용해 값을 저장한 후, 띄어쓰기, 탭, / 등의 문자를 이용해 값을 분리
      • 서울대 엑셀 파일의 경우, /를 이용해 시간과 공간 정보를 분리함
      • 정수를 복수 선택해야 하는 경우
      • 문자열의 경우 오류가 발생할 수 있음
